### 平台优势

```
# 实现从设计（xls表）一键生成微服务全部代码（前端代码+后端代码），并可直接编译运行。
# 实现了单体编码，自动微服务拆分和微服务部署。
# 完全不需要处理微服务之间的接口调试，不需要处理微服务升级时的版本匹配。
# 框架自动实现了日志记录、指标、跟踪、路由、测试、弹性伸缩等功能。
# 编译结果可单机运行，也可分布式运行，仅需使用不同运行指令即可：
    单进程：weaver single deploy weaver.toml
    多进程：weaver multi  deploy weaver.toml
    分布式：weaver kube   deploy config.yaml

```

### 主要功能

```
# 配置组件：全局数据统一配置，所有微服务模块均可象函数一样调用
# 鉴权组件: jwt鉴权和token发放
# 权限组件：casbin鉴权和api分配
# 角色组件：角色管理，角色菜单权限分配，角色api权限分配
# 用户组件：用户管理，用户的角色分配
# 菜单组件：菜单结构调整
# 字典组件：系统字典配置和管理
# 状态组件：显示系统的运行状态
# 路由组件：整个平台的路由分配映射
# 黑名单组件：管理外部访问用户,角色,ip(mac)
# 前端配置组件：调整前端页面列表，表单的显示内容，排列顺序,输入方式,能否输入等
# 文件服务组件：整个平台的文件上传下载和分类。作为文件服务器
# 访问记录组件：记录外部通过api访问的请求内容和返回内容
```

### 技术栈

```
# 后端：service weaver框架(微服务),gin框架(路由分配),golang版本>1.21
# 前端：VITE+VUE+JS+Element-Plus
# 数据库：mysql系列或postgresql系列
# 缓存：redis(或etcd),本地cache
# 分布式部署：k8s (开发时，可不需要，service weaver自带k8s)
```
